# Prefix statistics (TZUR Original display wordlists)

Generated by [`validation/prefix_stats.py`](../validation/prefix_stats.py). Regenerate with `python validation/prefix_stats.py --write`.

BIP-39's original 4-character-prefix property lets a wallet identify a word from its first four letters, which is what makes prefix autocomplete possible. It is achievable for English and most Latin-script lists but structurally infeasible for several scripts where short word stems dominate. The BIP draft (Display wordlist requirements SHOULD 1) treats 4-character prefix uniqueness as a SHOULD, not a MUST, and instructs wallets relying on prefix autocomplete to fall back to full-word matching wherever uniqueness is below 2048/2048. This report quantifies realized uniqueness per language so implementers know exactly when that fallback is required.

**"Character" means Unicode codepoint, not grapheme cluster.** For scripts that build syllables from combining sequences a codepoint prefix can split a grapheme, so these figures are a conservative lower bound on what a grapheme-aware implementation would achieve. Plan against these numbers, not against a more optimistic grapheme count.

**Bottom line:** 4-character prefix uniqueness holds for only one language (Korean); two more (Chinese Simplified, Chinese Traditional) and Japanese miss it by a single collision. Every other list falls short, so a wallet that ships display wordlists MUST NOT assume 4-character prefixes are unique and MUST fall back to full-word matching. Use the per-language figures below to decide whether to offer prefix autocomplete at all in a given language.
